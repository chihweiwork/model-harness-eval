```
The `task` tool allows you to delegate complex or large tasks to specialized "agent" instances that run in their own context windows. This keeps your main conversation clean while allowing the system to handle heavy processing or multi-step workflows autonomously.

Here is a breakdown of the available agent types and when to use each:

### 1. `explore` (Codebase Exploration & Research)
*   **Use case:** When you need to investigate a large codebase, understand how several different components interact, or find relevant files across many directories.
*   **Why choose this:** It uses a fast, lightweight model optimized for navigating code and performing broad information gathering without cluttering your main context with "discovery" steps.

### 2. `task` (Action-Oriented & Execution)
*   **Use case:** Running commands like `npm test`, `go build`, linting scripts, or installing dependencies.
*   **Why choose this:** It is designed for operations where you primarily care about the result (Success/Failure). It provides a concise summary if something succeeds but captures the full technical output if a command fails so you can debug it.

### 3. `general-purpose` (Complex Multi-step Logic)
*   **Use case:** Complex tasks that require high-level reasoning, such as "Refactor this entire module to use a different state management library" or "Implement a new API endpoint including validation and error handling."
*   **Why choose this:** It uses a high-capability model. Since it's in a separate window, it can perform multiple steps (edit, test, fix) without filling up your current prompt history.

### 4. `code-review` & `security-review` (Specialized Review)
*   **code_review:** Use this for reviewing diffs or specific code blocks to find logic errors and bugs. 
*   **security_review:** **Mandatory** use whenever the user explicitly asks to find security vulnerabilities. It is a specialized read-only agent focused on high-confidence security risks.

### 5. `research` (External & Deep Search)
*   **Use case:** When you need deep research involving external sources, technical documentation, or verifying information from across the web.
*   **Why choose this:** It is specifically built to fetch files and provide citations for its findings.

---

### Important Usage Guidelines:

*   **Modes**: 
    *   `background`: The agent runs in the background. You are notified when it finishes, but you can continue doing other things (or it can even wait for instructions) while it works.
    *   `sync`: Use this if the task is simple but requires an LLM's reasoning to process immediately before you can proceed with your next step.
*   **When NOT to use the `task` tool**: 
    To keep performance high, avoid using a sub-agent for things you can do directly:
    *   Reading a file you already know the path to (use `view`).
    *   Simple grep/glob searches (use `grep` or `glob` directly).
    *   Single command execution where you need the output immediately to decide your next move.
    *   Simple edits on known files (use `edit` or `create`).
```
