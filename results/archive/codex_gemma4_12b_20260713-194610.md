Here is a breakdown of what makes Dapper unique:

### 1. Key Characteristics
*   **Lightweight:** Unlike full ORMs (like Entity Framework), Dapper doesn't try to do everything. It doesn't have a built-in LINQ provider, complex change tracking, or an automated migration system. Instead, it focuses on one thing: **mapping database results to C# objects.**
*   **Speed:** Because it has so little overhead, Dapper is nearly as fast as using raw ADO.NET (the native .NET way to talk to databases). It achieves this by using "IL" (Intermediate Language) generation to map data types quickly at runtime.
*   **SQL-First approach:** Dapper encourages developers to write their own SQL. You provide the SQL query, and Dapper handles the tedious work of taking those results and turning them into a collection of objects (POCOs).

### 2. Core Features
*   **Mapping:** It automatically maps columns in your result set (e.g., `FirstName`, `Age`) to properties on your C# classes.
*   **Multi-Mapping:** It can map a single row of results into multiple objects simultaneously (useful for joins).
*   **Stored Procedures:** It has native support for executing stored procedures and returning results easily.
*   **Anonymous Types:** You can query the database and map directly to anonymous types or dynamic objects if you don't want to define a class first.
